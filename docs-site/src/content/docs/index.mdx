## O que o Workstate captura?

O Workstate foi desenhado para capturar tudo o que o controle de versão tradicional (Git) ignora:

- **Variáveis de Ambiente**: `.env`, `.env.local`, arquivos de configuração.
- **Configurações de IDE**: Projetos do VS Code (`.vscode/`), IntelliJ (`.idea/`), etc.
- **Bancos de Dados Locais**: Arquivos SQLite, dumps de banco local.
- **Dados de Desenvolvimento**: Dados de teste, mocks, assets locais.
